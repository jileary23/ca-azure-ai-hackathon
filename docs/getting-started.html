Quick Start

Three commands to go from zero to running. Mock mode means no Azure credentials needed.

Open in GitHub Codespace

Navigate to the repository on GitHub and launch a Codespace. Everything is pre-configured in .devcontainer/.

On the GitHub repo page:

① Click Code → ② Select Codespaces tab → ③ Create codespace on main

Install dependencies

One command installs both Python and Node.js dependencies across all projects.

bash 
npm run setup

Start the platform

Launches the core platform backend and frontend in mock mode — no cloud services needed.

bash 
npm start

Backend API

http://localhost:8000

Frontend UI

http://localhost:5173

Running an Accelerator

Each accelerator is independently runnable. Pick one to start, or spin them all up with Docker Compose.

A Run a single accelerator

bash 
# Run BenefitsCal Navigator (accelerator 001)
./scripts/run-accelerator.sh 001

# Run Multilingual Emergency Chatbot (accelerator 008)
./scripts/run-accelerator.sh 008

B Run all accelerators with Docker Compose

bash 
# Start all 8 accelerators
docker-compose -f docker-compose.accelerators.yml up

# Or rebuild and start
docker-compose -f docker-compose.accelerators.yml up --build

C Run a specific accelerator via Docker Compose

bash 
# Run just the Multilingual Emergency Chatbot
docker-compose -f docker-compose.accelerators.yml up accel-008-backend accel-008-frontend

Workshop Labs

A structured 8-lab curriculum that progresses from local development to full Azure deployment. Labs 00–03 require no Azure account.

Lab Topic Description Azure Required?
00 Environment Setup Configure your Codespace, install dependencies, verify mock services No
01 First Agent Build your first QueryAgent with intent detection and entity extraction No
02 RAG Pipeline Implement retrieval-augmented generation with domain knowledge bases No
03 Agent Orchestration Wire up the full 3-agent pipeline: Query → Router → Action No
04 Azure OpenAI Connect to Azure OpenAI for real LLM-powered responses Yes
05 Azure AI Search Replace mock search with Azure AI Search for production RAG Yes
06 Voice (Realtime API) Add voice input/output using Azure OpenAI Realtime API Yes
07 MCP Server Build a Model Context Protocol server for tool integration Yes

Tip: Labs are located in the labs/ directory. Each lab has its own README with step-by-step instructions. Start with Lab 00 to verify your environment.

Deploying to Azure

When you're ready to go beyond mock mode, deploy to Azure with the Azure Developer CLI (azd).

Authenticate with Azure

Sign in with your Azure account. This grants azd permission to provision and deploy resources.

bash 
azd auth login

Deploy everything

Provisions all Azure infrastructure (via Bicep) and deploys every service in one command.

bash 
azd up

Or deploy a single accelerator

Deploy just one accelerator's backend and frontend without touching others.

bash 
# Deploy accelerator 001 (backend + frontend)
./scripts/azd-deploy.sh 001

# Or deploy backend and frontend separately
azd deploy accel-001
azd deploy accel-001-fe

Cost note: Azure Container Apps use scale-to-zero, so idle accelerators cost nothing. See Cost Estimation for detailed breakdowns.

Environment Variables

Configuration is driven by environment variables. Only USE_MOCK_SERVICES is required for local development — everything else is for Azure mode.

Variable Required Description
USE_MOCK_SERVICES Always Set to true for local dev (no Azure), false for production
AZURE_OPENAI_ENDPOINT Azure mode Azure OpenAI endpoint URL
AZURE_OPENAI_API_KEY Azure mode Azure OpenAI API key (or use managed identity)
AZURE_SEARCH_ENDPOINT Azure mode Azure AI Search endpoint URL
AZURE_SEARCH_API_KEY Azure mode Azure AI Search API key (or use managed identity)

Example .env file

env 
# Local development — no Azure required
USE_MOCK_SERVICES=true

# Azure mode — uncomment and fill in when ready
# USE_MOCK_SERVICES=false
# AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/
# AZURE_OPENAI_API_KEY=your-key-here
# AZURE_SEARCH_ENDPOINT=https://your-search.search.windows.net
# AZURE_SEARCH_API_KEY=your-key-here

Security: Never commit .env files with API keys to version control. The .gitignore already excludes .env files. In production, use managed identity (DefaultAzureCredential) instead of API keys.
