GenAI Procurement Compliance Checker

Automated compliance scoring for AI vendor proposals against California's 18+ AI-specific laws and federal frameworks.

1 Problem Statement

Executive Order N-5-26 requires AI vendors to attest to content safety, bias governance, and civil rights protections before state agencies can procure their products. However, agencies lack standardized tools to evaluate vendor proposals against the growing body of 18+ California AI-specific laws and regulations. Manual review is slow, inconsistent, and error-prone—creating procurement bottlenecks and compliance risk across the state.

2 Solution Overview

Compliance Scoring

18-category severity-weighted scoring engine evaluates vendor attestations across content safety, bias, privacy, transparency, and civil rights dimensions.

Rule Matching

Automated matching against EO N-5-26, SB 53, and NIST frameworks with 20 built-in compliance rules and severity-weighted violation detection.

NIST AI RMF Classification

Risk classification across all four NIST AI RMF functions: Govern, Map, Measure, and Manage—providing a holistic risk posture view.

Gap Analysis

Actionable gap analysis with prioritized recommendations, helping agencies understand exactly what vendors need to address before procurement approval.

3 Architecture — 3-Agent Pipeline

Q

QueryAgent

Parses vendor attestation documents, extracts compliance claims, identifies regulatory categories, and filters sensitive procurement data.

R

RouterAgent

Routes claims to appropriate compliance rule sets (EO N-5-26, SB 53, NIST AI RMF), determines severity weights, and triggers escalation for high-risk gaps.

A

ActionAgent

Scores attestations across 18 categories, generates compliance reports with risk tiers, and produces gap analysis with remediation recommendations.

4 API Endpoints

Method Path Description
GET /health Health check
POST /api/chat Main chat interface
GET /api/status Service status
POST /api/attestations/analyze Analyze vendor attestation
GET /api/compliance/rules?source= List compliance rules
GET /api/compliance/report/{id} Detailed compliance report
GET /api/nist/classify?description= NIST AI RMF classification

5 Data Models

AttestationRequest

Input payload for vendor attestation analysis.

{
  "vendor_name": "string",
  "product_name": "string",
  "attestation_text": "string",
  "framework": "eo_n_5_26 | sb_53 | nist"
}

ComplianceScore

Severity-weighted scoring across 18 compliance categories.

{
  "overall_score": 0.0-1.0,
  "risk_tier": "low | medium | high | critical",
  "category_scores": { /* 18 categories */ },
  "gaps": ["string"],
  "recommendations": ["string"]
}

ComplianceRule

20 built-in rules mapped to CA regulations.

{
  "rule_id": "string",
  "source": "eo_n_5_26 | sb_53 | nist",
  "category": "string",
  "severity": "low | medium | high | critical",
  "description": "string"
}

NistClassification

NIST AI RMF risk classification across four functions.

{
  "govern": { "score": 0.0-1.0, "findings": [] },
  "map":    { "score": 0.0-1.0, "findings": [] },
  "measure":{ "score": 0.0-1.0, "findings": [] },
  "manage": { "score": 0.0-1.0, "findings": [] }
}

6 Quick Start

This accelerator is API-only (no frontend). Start the backend service using Docker Compose:

docker-compose -f docker-compose.accelerators.yml up accel-005-backend
Backend Port: 8005
Frontend: N/A — API only
