The 3-Agent Pipeline

Every accelerator follows a consistent three-stage processing pipeline. User queries flow through intent detection, intelligent routing, and domain-specific action — ensuring reliable, governed responses.

Data Flow

QueryAgent

The first line of defense. Parses natural language, extracts structured entities (names, case numbers, dates), classifies intent, and strips PII before data proceeds downstream. Ensures no sensitive information leaks into logs or AI model contexts.

RouterAgent

Intelligent traffic controller. Maps classified intents to the correct state agency, assigns priority levels (P1–P4), and triggers escalation workflows when human intervention is required. Respects cross-agency boundaries defined in the constitution.

ActionAgent

The workhorse. Executes RAG queries against domain-specific knowledge bases, creates support tickets, formats multilingual responses, and performs agency-specific actions — all while enforcing the output constitution.

Mock Mode vs Azure Mode

A single environment variable switches between local development and cloud-powered production. The API contracts stay identical — your code doesn't change.

LOCAL DEV

Mock Mode

USE_MOCK_SERVICES=true
  • ✓ In-memory mock services — zero cloud dependencies
  • ✓ Perfect for hackathon development & demos
  • ✓ Instant startup, no API keys needed
  • ✓ Deterministic responses for testing
PRODUCTION

Azure Mode

USE_MOCK_SERVICES=false
  • ✓ Azure OpenAI for intelligent responses
  • ✓ Azure AI Search for RAG retrieval
  • ✓ Cosmos DB for persistent state
  • ✓ Production-grade security & scale

Key insight: Both modes expose identical REST API contracts. You can develop and test locally in mock mode, then flip the switch to Azure for production — no code changes required. Labs 00–03 run entirely in mock mode.

./scripts/azd-deploy.sh all

Constitution-Driven Development

Every agent response is governed by a shared constitution — a machine-readable policy document that enforces California's AI compliance framework at runtime.

The Constitution

Located at shared/constitution.md, the constitution defines agent boundaries, output policies, escalation rules, and compliance requirements.

Agents reference the constitution before every response. It acts as a system-level guardrail that cannot be overridden by user prompts.

California AI Compliance Framework

All accelerators must comply with these mandates:

📋

EO N-12-23 — GenAI Executive Order

Guidelines for responsible use of generative AI by California state agencies. Requires risk assessments and human oversight.

🏛️

EO N-5-26 — AI Procurement

Vendor attestation requirements and procurement standards for AI systems used in state government.

⚖️

SB 53 — AI Safety Legislation

Safety testing and reporting requirements for frontier AI models deployed in public-facing applications.

🔒

CCPA / CPRA — Privacy

California Consumer Privacy Act and its amendment. Governs PII handling, data minimization, and consumer rights.
